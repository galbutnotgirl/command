User Guide

Set up prompt-centered macOS shortcuts for Claude, ChatGPT, or Codex: selected text, screenshots, typed notes, voice, clipboard history, and background CLI runs.

Choose Claude, ChatGPT, or Codex

Choose Default assistant in Settings -> Shortcuts. Custom Actions and triggers can override it. Claude uses Chat/Cowork/Code destinations; Codex uses configured workspace. Command never silently falls back to another provider.

Quick Reference Settings Reference Examples

Quick Start

  1. Install and launch Command. See Install Guide for first-time install details.
  2. Open menu-bar icon, then choose Settings.
  3. In Set Up, grant Accessibility. Add Screen Recording for screenshots and Microphone for dictation or voice custom actions.
  4. Open Shortcuts and confirm built-in Compose combinations you want.
  5. Select text in any app and try Selected text -> Existing chat or Selected text -> New chat.

For permission details and reset commands, see Permissions.

You can change prompt and trigger shortcuts from Settings -> Shortcuts. Dictation shortcuts live in Settings -> Dictation Settings.

Menu Bar

Command lives in the macOS menu bar. The menu is intentionally short.

Menu itemShows whenUse
Bound prompt/action shortcutsTrigger is enabled and has a key binding.Run that prompt/action without opening Settings.
Stop Dictation / Cancel DictationDictation is recording.End or cancel the current recording from the menu.
SettingsAlways.Open Settings; keyboard shortcut is Command-,.
Quit CommandAlways.Quit the app; keyboard shortcut is Command-Q.

Unbound combinations, disabled triggers, and auto-submit combinations with no shortcut do not appear in the menu. They stay editable in Settings -> Shortcuts.

Prompt Model

Command is organized around prompts. Each prompt can have many triggers, one default delivery, one default destination, and per-trigger overrides.

TriggersSelected text, screenshot, popup, voice.
DeliveryExisting chat, new chat, background.
DestinationDefault, Chat, Cowork, Code.
Inheritance— means trigger inherits prompt/action default.

Background Actions

Background delivery runs without opening Claude. Command renders prompt and sends it to claude -p. Runs appear under Command History -> Background with status, age, logs, retry, mark-failed, retention, and parsed result.

If final non-empty output line matches KEY=value, Command shows it in notification and Command History.

Legacy right-click Services are optional for source installs. If installed, Claude - To-Do is a background action: selected text wins when highlighted; otherwise Safari, Chrome, Brave, Chromium, and Arc send the current tab URL. Confirm the captured source, result, and log under Command History -> Background.

Only the last non-empty stdout line is parsed. The whole trimmed line must be KEY=value; prose containing TASK_ID=abc123 does not count. Common contracts are TASK_ID=<id> and ERROR=<reason>. Result text appears in the completion notification, Command History row, and diagnostic summary. Command does not run follow-up actions from that value yet.

StatusMeaningWhat to do
RunningLocal claude -p process has started.Wait, or use Command History if it becomes stalled.
SucceededCLI exited with code 0.Review result and log.
FailedCLI failed, command was missing, or user marked the run failed.Open log, fix prompt/settings, then Retry.
StalledRecord stayed running after process likely died.Mark failed, then Retry if the prompt is still valid.

Clipboard History

Open Settings -> Clipboard History to enable local clipboard history, set the shortcut, choose retention, pick a theme, or clear recent clips.

KeyAction
TypeSearch clips.
Up/DownMove selection.
ReturnPaste selected clip.
Command-ReturnPaste and keep picker open.
EscClose picker.

Clipboard history stays local. Secret-looking copies and copies from known password apps are skipped.

Dictation

Dictation is local and on-device through Parakeet TDT. Built-in Dictate shortcuts live in Dictation Settings. Voice custom actions live in Shortcuts.

Use Dictation Settings for Dictate -> Insert and Dictate -> Claude shortcuts. Use Shortcuts for voice triggers tied to prompt actions. If final words are missing, compare Dictation History raw text, processed text, and the sent command before filing a bug.

TabPurpose
HistoryRaw and processed dictation records.
CorrectionsMisheard -> correct replacement rules.
VocabularyProper nouns, product names, filler words.
Dictation SettingsModel, mic access, shortcuts, sounds.

Command History

Open Settings -> Command History to review recent sends and background runs. Retention defaults to seven days.

SectionIncludes
ForegroundExisting chat and New chat sends.
Backgroundclaude -p runs, logs, retry, and parsed result.

Use filters for All, Running, Succeeded, and Failed.

Import And Export

Open Settings -> About -> Import / Export. Export can include shortcuts, prompt settings, context rules, dictation vocabulary, background settings, and app preferences. Import previews sections and lets you keep, merge, or overwrite each section.

Privacy And Local Files

Command stores its own settings and history on this Mac. Background actions are different from local clipboard and dictation history: they run the local Claude CLI, so network access, file access, and tool use depend on your Claude CLI setup.

AreaLocation
Shortcuts and prompt text~/.claude/state/
Clipboard history~/.claude/state/cliphistory/
Dictation history and vocabulary~/Library/Application Support/DictationLab/
Command history~/Library/Application Support/claude-command/command-history/
Background runs~/Library/Application Support/claude-command/submissions/ and logs/

Review background prompts and CLI extra args before sharing exports. See Privacy for more detail.

Updating

Open Settings -> About to pick Alpha or Beta, check for updates, install available releases, and export/import settings. Stable is visible but unavailable until the first stable release exists. See Updates for manual alpha installs, backups, failed updates, and rollback.

Troubleshooting

ProblemFix
Hotkeys do nothingGrant Accessibility, then restart Command.
Shortcut keys conflictRebind prompt shortcuts in Shortcuts; rebind dictation shortcuts in Dictation Settings.
Screenshot failsGrant Screen Recording, then restart Command.
Dictation failsGrant Microphone; download model in Dictation Settings.
Background run failsOpen Command History -> Background and inspect log.

Need every symptom and log path? Read Troubleshooting. Need to file a bug, request a feature, or ask for help? Read Support. For vulnerabilities, exposed secrets, private logs, or sensitive diagnostics, read Security Policy.

Uninstall

For full removal steps, local data choices, and verification commands, see Uninstall.

For a source checkout that installed Quick Actions and a LaunchAgent:

./uninstall-quick-action.sh
launchctl bootout "gui/$(id -u)/com.claudecommand" 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.claudecommand.plist

Older alpha builds used a separate clipboard watcher LaunchAgent. Remove that legacy agent too if it exists.

launchctl bootout "gui/$(id -u)/com.claudecommand.clipwatch" 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.claudecommand.clipwatch.plist

Then remove ~/Applications/Command.app.
